## Overview

The Tuco AI API provides programmatic access to all platform features including lead management, availability checking, and message sending. Build custom integrations, automate workflows, and extend Tuco's capabilities to fit your needs.

<Card title="Base URL" icon="link">
  ```
  https://api.tuco.ai/v1
  ```
</Card>

## Authentication

All API requests require authentication using Clerk tokens in the Authorization header.

### Getting Your Token

<Steps>
  <Step title="Sign In">
    Log in to your Tuco AI account at [app.tuco.ai](https://app.tuco.ai)
  </Step>
  
  <Step title="Generate Token">
    Navigate to **Settings → API Keys** and generate a new token
  </Step>
  
  <Step title="Copy Token">
    Copy your token and store it securely (it won't be shown again)
  </Step>
</Steps>

### Using Your Token

Include your token in the Authorization header of every request:

```bash
curl https://api.tuco.ai/v1/leads \
  -H "Authorization: Bearer YOUR_CLERK_TOKEN" \
  -H "Content-Type: application/json"
```

<Warning>
**Keep your API token secure!** Never expose it in client-side code or public repositories.
</Warning>

## Request Format

All POST and PUT requests must include a JSON body with the `Content-Type: application/json` header.

### Headers

```json
{
  "Content-Type": "application/json",
  "Authorization": "Bearer YOUR_CLERK_TOKEN"
}
```

### Example Request

```bash
POST /api/leads
Content-Type: application/json
Authorization: Bearer clerk_token_xxxxx

{
  "leads": [
    {
      "firstName": "John",
      "lastName": "Doe",
      "email": "john@example.com",
      "phone": "+12025551234"
    }
  ],
  "listId": "507f1f77bcf86cd799439011"
}
```

## Response Format

All responses are returned in JSON format with consistent structure.

### Success Response

```json
{
  "success": true,
  "data": {
    // Response data here
  }
}
```

### Error Response

```json
{
  "success": false,
  "error": {
    "code": "INVALID_REQUEST",
    "message": "Missing required field: email"
  }
}
```

## HTTP Status Codes

| Code | Status | Description |
|------|--------|-------------|
| `200` | OK | Request succeeded |
| `201` | Created | Resource created successfully |
| `400` | Bad Request | Invalid request data |
| `401` | Unauthorized | Missing or invalid authentication |
| `403` | Forbidden | Insufficient permissions |
| `404` | Not Found | Resource not found |
| `429` | Too Many Requests | Rate limit exceeded |
| `500` | Internal Server Error | Server error occurred |

To ensure fair usage and platform stability, API requests are rate limited.

  <Card title="Premium Plan" icon="gauge-high">
    **1,000 requests/minute**

### Rate Limit Headers

Every response includes rate limit information:

```
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1634567890
```

<Tip>
When you exceed the rate limit, you'll receive a `429 Too Many Requests` response. Implement exponential backoff in your application to handle this gracefully.
</Tip>

## Pagination

List endpoints support pagination to handle large datasets efficiently.

### Query Parameters

<ParamField path="page" type="integer" default="1">
  Page number to retrieve
</ParamField>

<ParamField path="limit" type="integer" default="50">
  Number of items per page (max: 100)
</ParamField>

### Example Request

```bash
GET /api/leads?page=2&limit=25
```

### Paginated Response

```json
{
  "success": true,
  "data": [...],
  "pagination": {
    "page": 2,
    "limit": 25,
    "totalPages": 10,
    "totalItems": 250,
    "hasNext": true,
    "hasPrev": true
  }
}
```

## Code Examples

### JavaScript (Node.js)

```javascript
const axios = require('axios');

const tucoApi = axios.create({
  baseURL: 'https://api.tuco.ai/v1',
  headers: {
    'Authorization': 'Bearer YOUR_CLERK_TOKEN',
    'Content-Type': 'application/json'
  }
});

// Create a lead
async function createLead() {
  try {
    const response = await tucoApi.post('/leads', {
      leads: [{
        firstName: 'Jane',
        lastName: 'Smith',
        email: 'jane@example.com',
        phone: '+12025551234'
      }],
      listId: '507f1f77bcf86cd799439011'
    });
    console.log('Lead created:', response.data);
  } catch (error) {
    console.error('Error:', error.response.data);
  }
}
```

### Python

```python
import requests

BASE_URL = 'https://api.tuco.ai/v1'
HEADERS = {
    'Authorization': 'Bearer YOUR_CLERK_TOKEN',
    'Content-Type': 'application/json'
}

# Create a lead
def create_lead():
    payload = {
        'leads': [{
            'firstName': 'Jane',
            'lastName': 'Smith',
            'email': 'jane@example.com',
            'phone': '+12025551234'
        }],
        'listId': '507f1f77bcf86cd799439011'
    }
    
    response = requests.post(
        f'{BASE_URL}/leads',
        headers=HEADERS,
        json=payload
    )
    
    if response.status_code == 200:
        print('Lead created:', response.json())
    else:
        print('Error:', response.json())
```
