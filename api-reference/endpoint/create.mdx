## Overview

Create one or more leads in your Tuco AI workspace. Leads can be added to existing lists or new lists will be created automatically.

---

## Endpoint

```
POST /api/leads
```

---

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
  
  Example: `Bearer clerk_token_xxxxx`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

---

    <ParamField body="altPhone1" type="string">
      Alternate phone number 1
    </ParamField>
    
    <ParamField body="altPhone2" type="string">
      Alternate phone number 2
    </ParamField>
    
    <ParamField body="altPhone3" type="string">
      Alternate phone number 3
    </ParamField>
    
    <ParamField body="altEmail1" type="string">
      Alternate email address 1
    </ParamField>
    
    <ParamField body="altEmail2" type="string">
      Alternate email address 2
    </ParamField>
    
    <ParamField body="altEmail3" type="string">
      Alternate email address 3
    </ParamField>
    
    <ParamField body="companyName" type="string">
      Company or organization name
    </ParamField>
    
    <ParamField body="jobTitle" type="string">
      Job title or position
    </ParamField>
    
    <ParamField body="linkedinUrl" type="string">
      LinkedIn profile URL
    </ParamField>
    
    <ParamField body="notes" type="string">
      Additional notes or context
    </ParamField>
    
    <ParamField body="customFields" type="object">

<ParamField body="listId" type="string" required>

</ParamField>

<ParamField body="source" type="string">
  Source of the leads (for tracking)
  
  Examples: `csv_import`, `api`, `hubspot_sync`, `manual`
</ParamField>

## Error Codes

| Code | Status | Description | Solution |
|------|--------|-------------|----------|
| `400` | Bad Request | Missing required fields | Check that all required fields are provided |
| `400` | Bad Request | Invalid data format | Verify phone/email format is correct |
| `401` | Unauthorized | Invalid auth token | Check your Authorization header |
| `404` | Not Found | List not found | Verify the listId exists or will be created |
| `429` | Too Many Requests | Rate limit exceeded | Implement exponential backoff |
| `500` | Internal Server Error | Server error | Contact support if persists |

## Validation Rules

### Phone Numbers

<Info>
Phone numbers must be in E.164 format: `+[country code][number]`

**Examples:**
- US: `+12025551234`
- UK: `+447911123456`
- Australia: `+61412345678`
</Info>

**Auto-formatting:**
- 10 digits → `+1XXXXXXXXXX` (assumes US)
- 11 digits starting with 1 → `+1XXXXXXXXXX`

### Email Addresses

- Must contain `@` and `.`
- Automatically converted to lowercase
- Whitespace is trimmed

### Duplicate Detection

<Warning>
Duplicates are detected by email address. If a lead with the same email already exists in your workspace, the request will fail with a duplicate error.
</Warning>
