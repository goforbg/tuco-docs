## Overview

Delete one or more leads from your Tuco AI workspace. This action is permanent and cannot be undone.

<Warning>
**Permanent Deletion:** Deleted leads cannot be recovered. All associated message history and custom data will be lost.
</Warning>

---

## Delete Single Lead

Remove a specific lead by ID.

### Endpoint

```
DELETE /api/leads/{leadId}
```

### Path Parameters

<ParamField path="leadId" type="string" required>
  Unique identifier of the lead to delete
</ParamField>

---

## Delete Multiple Leads

Remove multiple leads in a single request.

### Endpoint

```
DELETE /api/leads
```

### Request Body

<ParamField body="leadIds" type="array" required>
  Array of lead IDs to delete
  
  **Example:** `["507f1f77bcf86cd799439011", "507f1f77bcf86cd799439012"]`
</ParamField>

---

## Delete Leads by Filter

Remove all leads matching specific criteria.

### Endpoint

```
DELETE /api/leads/filter
```

### Request Body

<ParamField body="listId" type="string">
  Delete all leads in a specific list
</ParamField>

<ParamField body="availabilityStatus" type="string">
  Delete all leads with specific availability status
  
  Values: `available`, `unavailable`, `error`, `no_active_line`
</ParamField>

<ParamField body="dateRange" type="object">
  Delete leads created within date range
  
  <Expandable title="Date Range Object">
    <ParamField body="start" type="datetime">
      Start date (ISO 8601 format)
    </ParamField>
    
    <ParamField body="end" type="datetime">
      End date (ISO 8601 format)
    </ParamField>
  </Expandable>
</ParamField>

<Warning>
**Use with Caution:** Bulk deletion by filter can remove many leads at once. Always verify your filters before executing.
</Warning>

```javascript Node.js
const axios = require('axios');

const deleteLead = async (leadId) => {

    console.error('Error:', error.response.data);
```

```python Python
import requests
import json

# Delete single lead
def delete_lead(lead_id):
    url = f'https://api.tuco.ai/v1/api/leads/{lead_id}'
    headers = {
        'Authorization': 'Bearer clerk_token_xxxxx'
    }
    
    response = requests.delete(url, headers=headers)
    
    if response.status_code == 200:
        print('Lead deleted:', response.json())
    else:
        print('Error:', response.json())

# Delete multiple leads
def delete_multiple_leads(lead_ids):
    url = 'https://api.tuco.ai/v1/api/leads'
    headers = {
        'Authorization': 'Bearer clerk_token_xxxxx',
        'Content-Type': 'application/json'
    }
    payload = {'leadIds': lead_ids}
    
    response = requests.delete(url, headers=headers, data=json.dumps(payload))
    
    if response.status_code == 200:
        print('Leads deleted:', response.json())
    else:
        print('Error:', response.json())

delete_lead('507f1f77bcf86cd799439011')
delete_multiple_leads(['507f...', '507f...'])
```

## Soft Delete vs Hard Delete

### Hard Delete (Default)

Permanently removes the lead and all associated data:

```bash
DELETE /api/leads/{leadId}
```

<Warning>
Cannot be recovered after deletion
</Warning>

### Soft Delete

Mark lead as deleted without removing from database:

```bash
PATCH /api/leads/{leadId}
Content-Type: application/json

{
  "isDeleted": true
}
```

<Info>
Soft-deleted leads are hidden from normal queries but can be restored if needed
</Info>

## Best Practices

<AccordionGroup>
  <Accordion title="Confirm Before Deletion" icon="triangle-exclamation">
    Always confirm with users before deleting leads, especially in bulk operations.
  </Accordion>
  
  <Accordion title="Export Before Deletion" icon="download">
    Export lead data before deletion if you might need it for compliance or backup purposes.
  </Accordion>
  
  <Accordion title="Use Soft Delete for Safety" icon="shield">
    Consider using soft delete (marking as deleted) instead of hard delete for important data.
  </Accordion>
  
  <Accordion title="Audit Trail" icon="list-check">
    Keep a log of deleted leads (ID, date, deleted by) for audit purposes.
  </Accordion>
  
  <Accordion title="Batch Operations" icon="layer-group">
    Delete multiple leads in batches rather than individual API calls for better performance.
  </Accordion>

## Recovery Options

<Warning>
**Hard deleted leads cannot be recovered** through the API. Contact support immediately if you accidentally delete critical data.
</Warning>

### Backup Recommendations

1. **Regular Exports:** Schedule daily/weekly exports of all lead data
2. **Database Backups:** Maintain external database backups
3. **Soft Delete First:** Use soft delete for 30 days before hard delete
4. **Audit Logs:** Keep detailed logs of all deletion operations

## Compliance & GDPR

### Right to Deletion

When a user requests data deletion under GDPR:

```bash
DELETE /api/leads/gdpr/{email}
```

This endpoint:
- Permanently deletes the lead
- Removes all associated message history
- Sends confirmation to the requester
- Logs the deletion for compliance

<Info>
GDPR deletion requests are processed immediately and cannot be undone. A compliance log is generated automatically.
</Info>
