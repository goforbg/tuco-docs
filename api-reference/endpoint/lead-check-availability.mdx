---
title: "Check iMessage Availability (Single Lead)"
api: "GET /api/leads/check-availability"
description: "Check if a specific lead is reachable via iMessage"
---

Check iMessage availability for a single lead using their stored phone numbers and emails. Updates the lead’s `availabilityStatus` field.

<Tip>
Don’t need a lead ID? Use [`POST /api/leads/check-availability`](/api-reference/endpoint/bulk-check-availability) with `{ "address": "+12025551234" }` to check any phone or email without creating a lead first.
</Tip>

## Authentication

```bash
Authorization: Bearer tuco_sk_xxxxxxxxxxxxx
```

## Query parameters

<ParamField query="id" type="string" required>The lead’s ID.</ParamField>

## Example

```bash
curl "https://app.tuco.ai/api/leads/check-availability?id=667f1f77bcf86cd799439012" \
  -H "Authorization: Bearer tuco_sk_xxxxxxxxxxxxx"
```

## Success response (200)

```json
{
  "success": true,
  "leadId": "667f1f77bcf86cd799439012",
  "available": true,
  "status": "available"
}
```

<ResponseField name="available" type="boolean">Whether the lead is reachable via iMessage.</ResponseField>
<ResponseField name="status" type="string">
  One of:
  - `"available"` — at least one stored address supports iMessage
  - `"unavailable"` — none of the stored addresses support iMessage
  - `"error"` — the check failed (temporary issue)
</ResponseField>

The same status is written back to the lead, so you can also read it via the Leads API.

## Error responses

| Status | When | Body |
|--------|------|------|
| `400` | Missing `id` parameter | `{ "error": "Lead ID is required" }` |
| `400` | Availability server not configured | `{ "error": "Availability check server not configured" }` |
| `401` | Invalid or missing API key | `{ "error": "Unauthorized" }` |
| `404` | Lead not found | `{ "error": "Lead not found" }` |
