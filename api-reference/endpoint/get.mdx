## Overview

Retrieve one or more leads from your Tuco AI workspace. Supports filtering, pagination, and search capabilities.

---

## Get All Leads

Retrieve a paginated list of all leads in your workspace.

### Endpoint

```
GET /api/leads
```

### Query Parameters

<ParamField query="page" type="integer" default="1">
  Page number to retrieve
</ParamField>

<ParamField query="limit" type="integer" default="50">
  Number of leads per page (max: 100)
</ParamField>

<ParamField query="listId" type="string">
  Filter leads by list ID
</ParamField>

<ParamField query="search" type="string">
  Search leads by name, email, or phone
</ParamField>

<ParamField query="availabilityStatus" type="string">
  Filter by iMessage availability status
  
  Values: `available`, `unavailable`, `checking`, `error`, `no_active_line`
</ParamField>

<ParamField query="sort" type="string" default="createdAt">
  Sort field
  
  Values: `createdAt`, `updatedAt`, `firstName`, `lastName`
</ParamField>

<ParamField query="order" type="string" default="desc">
  Sort order
  
  Values: `asc`, `desc`
</ParamField>

---

## Get Single Lead

Retrieve a specific lead by ID.

### Endpoint

```
GET /api/leads/{leadId}
```

### Path Parameters

<ParamField path="leadId" type="string" required>
  Unique identifier of the lead
</ParamField>

---

## Check Lead Availability

Check or re-check iMessage availability for a lead.

### Endpoint

```
GET /api/leads/check-availability?id={leadId}
```

### Query Parameters

<ParamField query="id" type="string" required>
  Lead ID to check availability for
</ParamField>

```javascript Node.js
const axios = require('axios');

const getLeads = async () => {
  try {
    const response = await axios.get('https://api.tuco.ai/v1/api/leads', {
      params: {
        page: 1,
        limit: 50,
        availabilityStatus: 'available'
      },
      headers: {
        'Authorization': 'Bearer clerk_token_xxxxx'
      }
    });
    
    console.log('Leads:', response.data);
    console.log('Pagination:', response.data.pagination);
  } catch (error) {
    console.error('Error:', error.response.data);
  }
};

getLeads();
```

```python Python
import requests

url = 'https://api.tuco.ai/v1/api/leads'
headers = {
    'Authorization': 'Bearer clerk_token_xxxxx'
}
params = {
    'page': 1,
    'limit': 50,
    'availabilityStatus': 'available'
}

response = requests.get(url, headers=headers, params=params)

if response.status_code == 200:
    data = response.json()
    print('Leads:', data['data'])
    print('Pagination:', data['pagination'])
else:
    print('Error:', response.json())
```

## Error Codes

| Code | Status | Description | Solution |
|------|--------|-------------|----------|
| `400` | Bad Request | Invalid query parameters | Check parameter formats |
| `401` | Unauthorized | Invalid auth token | Verify Authorization header |
| `404` | Not Found | Lead not found | Check lead ID is correct |
| `429` | Too Many Requests | Rate limit exceeded | Implement rate limiting |
| `500` | Internal Server Error | Server error | Contact support if persists |

## Filtering & Search

### By Availability Status

Filter leads by their iMessage availability:

```bash
GET /api/leads?availabilityStatus=available
```

### By List

Get all leads in a specific list:

```bash
GET /api/leads?listId=507f1f77bcf86cd799439011
```

### Search

Search across name, email, and phone fields:

```bash
GET /api/leads?search=john
```

<Tip>
**Combine filters** for more specific queries:

```bash
GET /api/leads?listId=abc123&availabilityStatus=available&search=tech
```
</Tip>

## Pagination

Navigate through large datasets efficiently:

### First Page
```bash
GET /api/leads?page=1&limit=25
```

### Next Page
```bash
GET /api/leads?page=2&limit=25
```

### Last Page
Use `totalPages` from response:
```bash
GET /api/leads?page=10&limit=25
```

<Info>
**Best Practice:** Start with smaller page sizes (25-50) and adjust based on your needs. Maximum limit is 100 items per page.
</Info>

## Best Practices

<AccordionGroup>
  <Accordion title="Use Pagination" icon="list-ol">
    Always use pagination when retrieving large datasets to avoid timeouts and improve performance.
  </Accordion>
  
  <Accordion title="Filter Server-Side" icon="filter">
    Use query parameters to filter data on the server rather than fetching everything and filtering client-side.
  </Accordion>
  
  <Accordion title="Cache Results" icon="database">
    Cache frequently accessed lead data to reduce API calls and improve application performance.
  </Accordion>
  
  <Accordion title="Handle Missing Fields" icon="shield-halved">
    Not all leads have all optional fields. Always check for field existence before using values.
  </Accordion>
</AccordionGroup>
